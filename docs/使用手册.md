# 曼克斯劳务派遣管理系统 — 使用手册

## 一、这是什么系统

把原来用 Excel 管的人员、工资、回款、日记账全部搬到网页上。几个人可以同时用，不用传文件、不用对版本，数据自动关联，改了什么都能查到。

---

## 二、谁会用到这个系统

系统里有三种账号，权限不同：

| 角色 | 能做什么 | 不能做什么 |
|------|---------|-----------|
| **一级员工** | 录入工资、返费数据 | 看不到公司整体数据，不能删数据 |
| **二级财务** | 查看全部数据，审核工资和返费，管理所有模块 | 不能删数据，不能创建账号 |
| **三级老板** | 系统全部权限 | 无限制 |

还有一个**系统管理员**（安装时创建），可以创建以上三种账号。

---

## 三、界面怎么看

打开网页后：

- **左边**是导航栏，业务模块分成三组：
  - 人员与用工（人员、考勤、合同）
  - 企业与合作（企业、岗位、回款、应收、开票）
  - 财务与薪资（工资、返费、审批、日记账、核对、利润）

- 点左边任意一个模块，右边就会显示对应的数据表格。

- **右上角**显示你的名字，旁边是退出登录按钮。

- **首页**（经营工作台）打开就能看到：当前在职多少人、合作企业多少家、本月收入/支出/利润、日记账多少笔、有什么预警、多少条待审批。

---

## 四、通用操作说明

所有模块的操作方式一样：

### 查看数据
表格默认每页显示 30 条，底部可以切换 30/50/100 条，点"上一页""下一页"翻页。

### 手工新增
每个模块表格第一行都有一个 **「+ 新增」** 按钮，点了之后出现一行输入框，填完点"保存"。如果该模块依赖其他模块（比如人员依赖企业），需要先把依赖的数据建好。

### 编辑
**点击表格中任意一行**，该行会变成编辑状态，改完点"保存"，不想改了点"取消"。

### 删除 / 作废
每行右边有操作按钮：
- 人员：点"离职"办理离职
- 合同：点"终止"结束合同
- 企业：点"停用"停止合作
- 其他模块：点"✕"作废该条记录

### 导入 Excel
每个模块右上角都有 **「导入」** 按钮。流程：
1. 点「导入」，选择电脑上的 Excel 文件（.xlsx）
2. 选工作表（如果 Excel 有多个 Sheet）
3. 点「整理并校验」，系统会检查每一行数据
4. 绿色"可写入"的行没问题，红色"需修正"的行有错误
5. 点「确认写入数据库」，正确的行进入正式数据，错误的不会写入

> 建议先下载模板或在已有数据上导出一份看看格式，再按同样格式准备导入文件。

### 导出 Excel
每个模块右上角都有 **「导出」** 按钮，点了会下载一个 Excel 文件，包含当前筛选条件下该模块的全部数据。

### 清空数据
如果导入错了或者想重新来，点 **「清空」** 按钮，选择日期范围后可以批量清除该模块的数据。**这个操作不可撤销，请确认后再点。**

## 五、各模块说明

### 1. 企业管理

**干什么的**：录入跟你合作的用工企业信息。这是最先要建的基础数据，后面岗位和人员都依赖企业。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 企业名称 | 不能重复 |
| 联系人 / 电话 | 用工企业对接人 |
| 合作状态 | 正常合作 / 暂停合作 / 终止合作 |
| 合作期限 | 合作的起止时间 |
| 默认回款天数 | 给这家企业设一个默认的回款期限，比如 30 天，方便后面判断逾期 |

**操作**：新增、编辑、停用、导入、导出、清空。停用后合作状态变为"终止合作"，数据还在。

---

### 2. 岗位管理

**干什么的**：给每家企业创建岗位，比如"A公司—普工""B公司—焊工"。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 所属企业 | 岗位属于哪家公司 |
| 岗位名称 | 岗位叫什么 |
| 日单价 | 这个岗位每天的工价（元），比如 280 |
| 需求人数 | 需要招多少人 |
| 状态 | 招聘中 / 已满 / 已关闭 |

**操作**：新增、编辑、关闭、导入、导出、清空。岗位关闭后不能再关联人员。

---

### 3. 人员管理

**干什么的**：管理派遣员工的基本档案。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 姓名 | 员工姓名 |
| 身份证号 | 18 位，系统加密存储，只有导出时能看到完整号码 |
| 手机号 | 11 位，1 开头 |
| 性别 | 男 / 女 |
| 企业 / 岗位 | 这个员工被派到哪家公司、做什么岗位 |
| 入职日期 | 哪一天开始上班 |
| 状态 | 在职 / 离职 |

**点击人员姓名**可以打开详情面板，看到这个人的所有合同、考勤、工资记录。

**操作**：新增、编辑、离职、导入、导出、清空。点"离职"后状态变为离职，不能再被关联到新的考勤和工资。

**搜索**：顶部的搜索框可以按姓名或手机号搜索。

---

### 4. 合同管理

**干什么的**：管理员工的劳动合同，系统会自动预警到期和未签。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 员工 | 跟谁签 |
| 合同编号 | 合同编号，可不填 |
| 签订日期 | 哪天签的 |
| 起止日期 | 合同有效期 |
| 状态 | 生效中 / 已到期 / 已终止 |

**预警规则**：
- 合同到期前 15 天，首页预警列表会出现提醒
- 员工入职超过 20 天还没签合同，也会出现预警

**操作**：新增、编辑、终止、导入、导出、清空。点"终止"后合同状态变为已终止。

---

### 5. 考勤管理

**干什么的**：记录每个员工每天的出勤情况，异常出勤可以记扣款。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 日期 | 考勤日期 |
| 人员 | 哪个员工 |
| 状态 | 正常 / 迟到 / 旷工 / 请假 |
| 工时 | 当天工作了多少小时 |
| 扣款 | 迟到、旷工等异常需要扣多少钱 |
| 备注 | 额外说明 |

**操作**：新增、编辑、删除、导入、导出、清空。

---

### 6. 工资发放

**干什么的**：按员工、按月份记录工资。**需要审批后才能生效**。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 员工 | 给谁发 |
| 工资月份 | 发的哪个月的工资，比如 2026-07 |
| 发薪日期 | 实际哪天发的 |
| 基本工资 | 基本工资 |
| 津贴 | 各种补贴 |
| 扣款 | 扣除的金额 |
| 实发金额 | 系统自动算：基本工资 + 津贴 - 扣款 |
| 备注 | 额外说明 |

**审批流程**（重要）：
1. 员工录入工资 → 状态为"待财务审核"
2. 财务审核通过 → 状态变为"待老板确认"
3. 老板确认 → 状态变为"已确认"，同时**自动在日记账里记一笔支出**

**操作**：新增、编辑、删除、导入、导出、清空。审批按钮会在非"已确认"状态下显示。

---

### 7. 代招返费

**干什么的**：记录为用工企业代招员工后，企业支付的招聘服务费。**同样需要审批**。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 企业 | 哪家企业给的返费 |
| 关联员工 | 跟哪个员工相关（可不填） |
| 返费日期 | 哪天收到的 |
| 金额 | 返费金额 |
| 人数 | 涉及多少个人 |
| 备注 | 额外说明 |

**审批流程和工资一样**：员工录入 → 财务审核 → 老板确认 → 自动记日记账。

**操作**：新增、编辑、删除、导入、导出、清空。

---

### 8. 回款管理

**干什么的**：记录用工企业给你打的款。有两种付款方式。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 企业 | 哪家公司回的款 |
| 回款日期 | 哪天到账的 |
| 金额 | 回款金额 |
| 付款方式 | **直接给付**（现金或转账）或 **银行承兑**（银行承兑汇票） |
| 承兑到期日 | 选了"银行承兑"才需要填，汇票到哪天才能兑付 |
| 银行流水 | 银行流水号（可不填） |
| 分配应收ID | 这笔回款对应哪笔应收记录（可不填） |
| 分配金额 | 分配多少到这笔记应收（可不填） |
| 备注 | 额外说明 |

**银行承兑说明**：银行承兑汇票不是马上到账的，有一个固定的到期日，到期后才能兑付。所以选了银行承兑必须填到期日。

**操作**：新增、编辑、删除、导入、导出、清空。

---

### 9. 应收管理

**干什么的**：记录应收的款项（预计什么时候收回多少），和回款管理配合使用。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 企业 | 哪家欠的 |
| 预计回款日 | 预计哪天收回 |
| 应收金额 | 应该收多少 |
| 已收金额 | 已经收了多少 |
| 状态 | 待回款 / 部分回款 / 已回款 |

**回款逾期**：如果过了预计回款日还没收完，首页会出现逾期预警。

**操作**：新增、编辑、删除、导出。

---

### 10. 开票管理

**干什么的**：记录给企业开的发票，发票号不能重复。

**字段说明**：
| 字段 | 说明 |
|------|------|
| 企业 | 给哪家开的 |
| 发票编号 | 发票号，不能重复 |
| 开票日期 | 哪天开的 |
| 金额 | 发票金额 |
| 备注 | 额外说明 |

**操作**：新增、编辑、删除、导入、导出。

---

### 11. 日记账

**干什么的**：公司所有资金进出的明细账，相当于银行的流水账。里面包含自动生成的和手工录入的。

**主要功能**：
- **筛选**：可以按"收入"/"支出"、"现金"/"银行"、日期范围筛选
- **搜索**：按摘要或付款方式搜索
- **汇总**：表格上方实时显示筛选范围内的收入合计、支出合计、净流入

**数据来源**：
- 工资确认后 → 自动生成一笔支出
- 返费确认后 → 自动生成一笔支出
- 回款录入后 → 自动生成一笔收入
- 也可以手工点「+ 新增」录入（比如银行手续费、利息等）

**来源列**：显示"导入"说明是 Excel 导入的，"手工"说明是手工新增或系统自动生成的。

**操作**：新增、编辑、作废、导入、导出、清空。点"✕"作废后该笔记录不计入汇总，但数据还在。

---

### 12. 审批中心

**干什么的**：集中查看所有待审批的工资和返费。不用逐个模块去找。

**流程说明**：
- 员工在工资或返费模块录入数据后，状态为"待财务审核"
- 财务在审批中心看到后，点"审核通过"，状态变为"待老板确认"
- 老板在审批中心看到后，点"确认"，状态变为"已确认"，同时自动写入日记账

**谁能审批**：只有二级财务和三级老板能审批。一级员工点了会提示没有权限。

---

### 13. 核对中心

**干什么的**：自动比对日记账里的金额和各业务模块的金额是否一致。

选择日期范围，点"核对"，系统会检查：
- 日记账里的收入是否等于回款确认的金额
- 日记账里的支出是否等于工资 + 返费确认的金额

如果有差异，会逐条列出差异明细。全部一致则显示"核对通过"。

---

### 14. 利润核算

**干什么的**：按月查看公司的盈利情况。

**计算方式**：收入（日记账收入） - 支出（日记账支出）= 净利润

选择月份范围点"查询"，表格显示每个月的收入、支出、净利润。净利润为正显示绿色，为负显示红色。

---

### 15. AI 顾问

**干什么的**：用自然语言问系统里的数据，不用自己手动查表。

**怎么用**：
- 在底部输入框打字提问，比如"本月利润多少""哪些回款逾期了""哪家企业回款最多"
- 点击快捷提问按钮可以快速提问
- AI 会自动查询系统数据然后回答

**注意**：AI 只读数据，不会修改任何数据。

---

### 16. 设置（账号管理）

**干什么的**：创建系统账号、分配角色。只有管理员能进。

创建账号时需要填：姓名、登录账号、密码（至少 10 位）、选择角色。创建后可以随时修改角色。

---

## 六、操作顺序建议

第一次使用建议按以下顺序：

1. 企业 → 建好合作企业
2. 岗位 → 给每家企业建岗位
3. 人员 → 录入派遣员工（或导入 Excel）
4. 合同 → 给员工录入合同
5. 考勤 → 每天记录出勤
6. 工资 → 每月发薪时录入，走审批
7. 回款 / 应收 → 记录企业回款
8. 日记账 → 日常查看资金流水

这个顺序是因为后面模块依赖前面的数据：岗位依赖企业、人员依赖企业和岗位、合同依赖人员、工资依赖人员。

---

## 七、注意事项

- **金额精确到分**，系统内部用分存储，不会有四舍五入误差
- **身份证加密存储**，数据库里是密文，只有导出时能看完整号码
- **财务数据一旦确认不可修改**，需要作废后重新录入
- **操作日志自动记录**，谁在什么时候改了什么都有记录，不可删除
- **清空操作不可撤销**，请确认后再执行
