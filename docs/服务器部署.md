# 阿里云 ECS 部署（ACR + Xshell + Xftp）

本项目部署地址按现有配置为 `https://catal.online/manx/`。后端镜像存放于阿里云 ACR，前端作为 Nginx 静态文件部署。

## 一、本地构建并推送后端镜像

在项目根目录执行：

```powershell
docker login --username=你的ACR登录名 crpi-zapnyuqlrws037uw.cn-shanghai.personal.cr.aliyuncs.com
docker build -t crpi-zapnyuqlrws037uw.cn-shanghai.personal.cr.aliyuncs.com/chemtutor/manx-backend:v1 .
docker push crpi-zapnyuqlrws037uw.cn-shanghai.personal.cr.aliyuncs.com/chemtutor/manx-backend:v1
```

ACR 密码是“访问凭证”页面设置的 Registry 独立密码，不是阿里云控制台登录密码。

以后更新建议递增标签，例如 `v2`、`v3`，并同步修改服务器 `.env` 中的 `MANX_IMAGE`，不要长期覆盖同一个标签。

## 二、本地构建前端

```powershell
cd frontend
$env:VITE_BASE='/manx/'
$env:VITE_API_PREFIX='/api/manx'
npm run build
```

构建结果在 `frontend/dist`。

## 三、Xftp 上传文件

假设服务器部署目录是 `/opt/chem`。先在 Xshell 创建目录：

```bash
sudo mkdir -p /opt/chem/manx_frontend/dist /opt/chem/manx_data/{pgdata,uploads,storage}
sudo chown -R $USER:$USER /opt/chem
```

使用 Xftp 上传：

- `frontend/dist/` 内全部文件 → `/opt/chem/manx_frontend/dist/`
- `deploy/docker-compose.yml` → `/opt/chem/docker-compose.yml.new`
- `deploy/nginx.conf` → `/opt/chem/nginx/nginx.conf.new`
- `backend/sql/schema_v1.sql` → `/opt/chem/schema_v1.sql`
- `deploy/.env.example` → `/opt/chem/.env.example`

不要直接覆盖原 Compose 和 Nginx。先备份、对比：

```bash
cd /opt/chem
cp docker-compose.yml docker-compose.yml.bak-$(date +%F-%H%M)
cp nginx/nginx.conf nginx/nginx.conf.bak-$(date +%F-%H%M)
diff -u docker-compose.yml docker-compose.yml.new || true
diff -u nginx/nginx.conf nginx/nginx.conf.new || true
```

确认 `.new` 文件保留了服务器原有服务和路径后再替换：

```bash
mv docker-compose.yml.new docker-compose.yml
mv nginx/nginx.conf.new nginx/nginx.conf
```

## 四、服务器环境变量

```bash
cd /opt/chem
cp .env.example .env
chmod 600 .env
```

生成随机值：

```bash
openssl rand -hex 24
openssl rand -base64 32 | tr '+/' '-_'
openssl rand -hex 32
```

依次用于 `MANX_DB_PASSWORD`、`DATA_ENCRYPTION_KEY`、`AUTH_SECRET`。编辑：

```bash
vi .env
```

不要使用项目历史中出现过的数据库密码、AI Key 或密钥；它们应在对应控制台重置。

## 五、安装和检查 Docker

```bash
docker --version
docker compose version
```

如果没有安装，按服务器系统使用阿里云 ECS 官方 Docker/Compose 安装文档。安装后启动并设置开机启动：

```bash
sudo systemctl enable --now docker
```

## 六、服务器登录 ACR 并启动

```bash
docker login --username=你的ACR登录名 crpi-zapnyuqlrws037uw.cn-shanghai.personal.cr.aliyuncs.com
cd /opt/chem
docker compose config
docker compose pull manx-backend
docker compose up -d manx-db manx-backend
docker compose ps
```

首次创建数据库时，`schema_v1.sql` 会由 PostgreSQL 自动执行。如果 `manx_data/pgdata` 已存在但尚无业务表，手工执行：

```bash
docker exec -i manx-db psql -U manx -d manx_hr < schema_v1.sql
```

如果 `.env` 修改了数据库用户名或数据库名，将命令中的 `manx`、`manx_hr` 替换为实际值。

## 七、检查并重载 Nginx

```bash
docker compose up -d nginx
docker exec chem-nginx nginx -t
docker exec chem-nginx nginx -s reload
```

确保阿里云安全组开放 TCP 80、443；不要向公网开放 PostgreSQL 5432。

## 八、验收

```bash
docker compose ps
docker logs --tail 100 manx-backend
docker logs --tail 100 manx-db
curl -I https://catal.online/manx/
curl https://catal.online/api/manx/api/health
curl https://catal.online/api/manx/api/health/db
```

浏览器访问 `https://catal.online/manx/`。首次使用会显示创建管理员页面。

验收业务：新增企业 → 岗位 → 人员 → 合同/考勤；导入日记账；退出后重新登录；刷新页面确认数据仍存在。

## 九、以后更新

后端：构建并推送新标签，修改 `.env` 的 `MANX_IMAGE`，然后：

```bash
docker compose pull manx-backend
docker compose up -d manx-backend
docker logs -f --tail 100 manx-backend
```

前端：重新执行第二步，用 Xftp 替换 `/opt/chem/manx_frontend/dist/` 内容，再刷新浏览器。

## 十、备份

```bash
mkdir -p /opt/chem/backups
docker exec manx-db pg_dump -U manx -d manx_hr -Fc > /opt/chem/backups/manx-$(date +%F-%H%M).dump
tar -czf /opt/chem/backups/manx-files-$(date +%F-%H%M).tar.gz /opt/chem/manx_data/uploads /opt/chem/manx_data/storage
```

数据库备份和上传文件备份必须同时保留，并定期下载到服务器以外的位置。
