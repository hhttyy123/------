# 产品优化计划 — 让导入顺手

> 版本：1.0 | 日期：2026-06-18 | 状态：待执行

---

## 目标

把"导入工具"变成用户真正想用的东西。核心原则三条：

1. **AI 自动完成的事不向用户汇报**
2. **上传后先看原始数据，再决定要不要分析**
3. **只把搞不定的问题亮出来，映射完美的折叠掉**

---

## 改什么

### 一、上传 → 预览先行

```
改前：拖入文件 → 自动进分析 → loading → 映射表
改后：拖入文件 → 立即看到原始表格 → 用户点 [开始分析] → loading → 结果
```

**后端**：加一个 `GET /api/v1/preview/{upload_id}` 接口，纯 pandas 读前 50 行返回

**前端**：上传完成后渲染一个原始数据表格（sticky 表头，横向滚动），底部一个大按钮「开始分析」

### 二、分析结果：展示整理后的数据

```
改前：技术映射表（module_key, field_key, confidence...）
改后：干净的数据表格 + 只有问题列亮出来
```

### 三、文件指纹匹配

同一格式的 Excel 第二次上传时，直接用上次的映射方案，跳过 AI：

```python
# 文件指纹 = 列名序列的 hash
fingerprint = hash("|".join(sorted(column_names)))
# 存在 → 直接用缓存的映射
# 不存在 → 调 Agent，分析完存入缓存
```

**收益**：秒出结果 + 省 token + 结果稳定

## 不改什么

- Agent 底层逻辑不变（还是发 Schema Registry + 样本给 DeepSeek）
- Schema Registry 结构不变
- 值转换逻辑不变
- 后端 API 三个核心端点不变（只加一个预览端点）

---

## 实施步骤

### 第 1 步：后端加预览接口
- `GET /api/v1/preview/{upload_id}` → 返回原始表头 + 前 50 行数据
- 同时返回 `file_fingerprint`（列名 hash），前端判断是否有缓存映射

### 第 2 步：前端改上传页
- 上传完成 → 调 preview 接口 → 渲染原始数据大表格
- 表格上方：文件名、Sheet 名、行列数
- 表格下方：大号「开始分析」按钮

### 第 3 步：前端改分析结果页
- 主体：整理后的数据表格（标准字段名作表头，转换后的值）
- 折叠区：「N 列自动映射完成」（点击展开看详情）
- 高亮区：问题列（置信度 < 0.7 或无法映射），每列一个下拉选择框
- 底部：「确认导入」按钮

### 第 4 步：加文件指纹缓存
- 后端 `data/storage/_fingerprint_cache.json`
- 格式：`{"fingerprint": {"module": "employee", "mappings": {...}, "saved_at": "..."}}`
- 分析前先查缓存，命中直接返回

### 第 5 步：调通验证
- 用仓库里两份 Excel 走完整链路
- 验证预览、分析、缓存命中

```
用户拖入 Excel

```
