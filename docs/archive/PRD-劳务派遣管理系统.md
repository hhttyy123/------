# 劳务派遣管理系统 — 产品需求文档 (PRD)

> 版本：3.0 | 日期：2026-06-15

---

## 1. 项目背景与问题

劳务派遣公司的日常运营涉及一套完整的业务流程：管理派遣员工、对接用工企业、核算工资、跟踪回款、处理代招返费、记录收支日记账、定期核算利润。目前这些工作全部依赖 Excel 手工完成，存在以下问题：

1. **数据分散，无法联动**：员工花名册、考勤表、工资表、回款表、日记账分散在独立文件中。发薪时要手动跨表查找员工信息、考勤数据、企业单价，效率低且易出错。
2. **多人协作困难**：老板、财务、员工无法同时操作同一份数据，只能通过文件传阅，版本混乱，容易出现覆盖或丢失。
3. **财务核算工作量巨大**：每月利润核算需要手工汇总工资支出、返费支出、企业回款、其他收支，再与日记账逐条对账，耗时长且容易遗漏。
4. **缺乏风险预警**：合同到期、回款逾期、入职超期未签合同等风险完全靠人脑记忆，没有自动化提醒机制。
5. **操作无法追溯**：无法知道谁在什么时间修改了什么数据，出现问题时难以定位责任。
6. **Excel 导入门槛高**：从不同来源拿到的 Excel 表头命名不统一（如"性别"vs"男/女"、"在职状态"vs"员工状态"），导致导入失败，需要人工逐列调整格式。

**目标**：用一个统一的系统替代所有零散的 Excel 文件，实现数据集中管理、业务自动联动、风险实时预警。

---

## 2. 用户角色

| 角色 | 职责范围 |
|------|---------|
| **员工** | 录入发薪数据、代招返费数据 |
| **财务** | 查看全部数据；审核录入内容；管理所有业务模块；创建新用户 |
| **老板** | 全部权限；确认关键操作；删除数据；系统最高管理者 |

权限规则：
- 老板拥有系统所有权限
- 财务拥有除删除数据和创建用户之外的全部权限
- 员工仅能录入数据，不能查看全局统计

## 3. 功能需求

### 3.1 员工管理

**3.1.1 员工信息维护**
- 录入员工全量信息：姓名、身份证号、手机号、性别、地址、入职日期
- 员工必须关联所属企业和所在岗位（一个员工属于一个企业的一个岗位）
- 支持编辑员工信息（岗位变动、状态变更等）
- 支持删除已离职员工（需老板权限，操作需确认）

**3.1.2 员工查询与筛选**
- 支持按姓名、手机号、所属企业、所在岗位、在职状态组合筛选
- 列表展示合同签订状态（已签/未签、签订日期、到期日期）
- 支持分页浏览

**3.1.3 员工数据导入导出**
- 支持下载 Excel 模板，按规范格式准备数据
- 支持从 Excel 批量导入员工数据
- 支持将员工数据导出为 Excel

**3.1.4 数据校验规则**
- 身份证号：18 位，需符合格式校验
- 手机号：11 位，1 开头
- 性别：界面显示中文（男/女），系统内统一存储
- 入职日期不能晚于当前日期

### 3.2 考勤管理

**3.2.1 考勤记录**
- 逐条记录员工每日出勤情况：出勤日期、出勤状态、工时
- 出勤状态包括：正常出勤、迟到、旷工、请假
- 异常出勤（迟到、旷工、缺勤）自动标记

**3.2.2 考勤与工资联动**
- 考勤异常自动关联工资扣款
- 迟到：按规则计算扣款金额
- 旷工：按规则计算扣款金额
- 请假：按规则计算扣款金额
- 发薪时自动汇总当月考勤扣款总额

**3.2.3 考勤查询与导入导出**
- 支持按员工、日期范围、出勤状态筛选
- 支持批量导入考勤 Excel
- 支持导出考勤数据为 Excel（用于与用工企业核对）

### 3.3 合同管理

**3.3.1 合同全生命周期**
- 创建合同记录，类型包括：初始签订、续签、终止
- 记录签订日期、合同起始日期、合同截止日期
- 支持上传合同模板文件和已签扫描件（电子归档）
- 合同状态：生效中、已到期、已终止

**3.3.2 合同预警**
- 合同到期提醒：到期前 15 天自动预警
- 入职未签合同提醒：入职超过 20 天仍未签订合同的员工自动预警
- 预警信息需展示：员工姓名、所属企业、到期/逾期天数

**3.3.3 合同查询**
- 支持按员工、合同类型、合同状态筛选
- 支持查看某员工的所有合同历史（含已终止的旧合同）

### 3.4 企业管理

**3.4.1 企业信息维护**
- 录入合作企业全量信息：企业名称、联系人、联系电话、地址、营业执照号
- 支持上传企业资质文件（营业执照等）
- 管理合作状态：正常合作、暂停合作、终止合作
- 记录合作起止日期

**3.4.2 企业统计视图**
- 查看企业下所有关联岗位
- 查看企业维度统计：当前在职人数、累计发薪总额、累计回款总额
- 企业资质到期提醒

**3.4.3 企业查询**
- 支持按企业名称、合作状态筛选
- 支持查看企业详情（聚合信息+岗位+统计数据）

### 3.5 岗位管理

**3.5.1 岗位维护**
- 为企业创建岗位：岗位名称、岗位描述、日单价、需求人数
- 编辑岗位信息（调整日单价、需求人数等）
- 岗位状态自动更新：招聘中（在岗 < 需求）、已满（在岗 = 需求）、已关闭

**3.5.2 岗位查询**
- 支持按所属企业筛选岗位
- 展示每个岗位的当前在岗人数 vs 需求人数

### 3.6 工资发放

**3.6.1 工资录入与构成**
- 按员工、按月份录入工资发放记录
- 工资构成必须拆分：基本工资 + 津贴 - 扣款 = 实发金额
- 扣款需关联考勤异常数据（自动汇总）
- 每个员工每月只能有一条工资记录

**3.6.2 工资状态管理**
- 工资状态：待发放、已发放
- 标记已发放后不可再修改（或需特殊权限）
- 发放后自动记入日记账（支出）

**3.6.3 工资查询与汇总**
- 查看某员工的所有历史工资记录
- 月度工资汇总：总人数、总基本工资、总津贴、总扣款、总实发金额
- 支持批量导入工资 Excel
- 支持导出工资数据为 Excel

**3.6.4 员工自助查询**
- 员工可查看本人的工资明细
- 权限隔离：员工只能看自己的，不能看他人

### 3.7 代招返费

**3.7.1 返费录入**
- 按企业录入返费记录：日期、金额、涉及人数
- 返费属于公司为用工企业代招员工的服务收入

**3.7.2 返费查询与汇总**
- 月度返费汇总（按企业维度）
- 返费自动记入日记账（支出端，与工资同样属于用工成本）
- 支持批量导入返费 Excel
- 支持导出返费数据为 Excel（用于与用工企业对账）

### 3.8 财务记录

**3.8.1 收支记录**
- 记录公司账户所有资金进出明细
- 每条记录包含：日期、类型（收入/支出）、类别、金额、所属企业、备注
- 支出类别包括：工资发放、代招返费、企业回款、办公费用、差旅费、其他

**3.8.2 收支汇总**
- 每日收支汇总
- 月度收支汇总
- 支持按日期、类型、类别、企业组合筛选

### 3.9 开票管理

**3.9.1 开票记录**
- 按企业录入开票信息：开票日期、发票号、金额、备注
- 发票号必须唯一，不可重复

**3.9.2 开票与回款关联**
- 每张开票可关联对应的回款记录
- 月度开票汇总（含未回款金额）
- 便于追踪"已开票但未回款"的情况

### 3.10 回款管理

**3.10.1 回款记录**
- 按企业录入回款记录：预计回款日期、实际回款日期、金额、付款方式
- 付款方式区分：
  - **直接给付**：现金或转账直接到账
  - **银行承兑**：银行承兑汇票，需额外记录承兑到期日

**3.10.2 逾期管理**
- 自动判断回款是否逾期（当前日期 > 预计回款日期 且 未到账）
- 自动计算逾期天数
- 逾期待办列表：展示所有逾期记录、逾期天数、逾期金额
- 支持自定义逾期预警阈值（7天/15天/30天），按企业或全局配置

**3.10.3 回款跟进**
- 每条回款记录支持添加跟进备注（可追加多条）
- 用于记录催收沟通情况

**3.10.4 回款汇总**
- 回款到账后自动记入日记账（收入）
- 月度回款汇总（按企业维度）

### 3.11 日记账

**3.11.1 日记账内容**
- 按日期记录所有资金进出明细（类似银行流水账）
- 条目来源区分：
  - **自动同步**：工资发放、返费支出、回款到账、财务记录自动写入日记账
  - **手动录入**：用户手工添加的条目（如银行手续费、利息等）

**3.11.2 日记账查询**
- 支持按日期（日/月/年/自定义时间段）筛选
- 支持按来源类型筛选（工资/返费/回款/财务/手动）
- 每日/月度/年度收支汇总

**3.11.3 日记账对账**
- 系统自动比对：日记账总额 vs 各财务模块汇总
- 不一致时触发提醒，并标注差异明细

**3.11.4 日记账导入导出**
- 支持从 Excel（如银行流水）批量导入日记账
- 支持导出日记账为 Excel（用于交由会计处理）

### 3.12 利润核算

**3.12.1 月度利润计算**
- 一键计算某月净利润，系统自动汇总：
  - **收入**：当月回款到账总额
  - **工资支出**：当月已发放工资总额
  - **返费支出**：当月返费总额
  - **其他支出**：当月其他财务支出总额
- 公式：净利润 = 收入 - 工资支出 - 返费支出 - 其他支出

**3.12.2 手工调整**
- 支持对自动计算结果进行手工修正
- 修正需留痕（记录修正前后值和修正原因）

**3.12.3 多维度汇总**
- 季度利润汇总（Q1/Q2/Q3/Q4）
- 年度利润汇总
- 导出利润核算数据为 Excel

**3.12.4 经营概览**
- 首页 Dashboard 展示核心指标：当前在职员工数、本月净利润、待处理预警数、本月回款金额
- 目标：3 秒看懂当前经营状况

### 3.13 AI 智能导入

**3.13.1 智能识别**
- 用户上传任意格式的 Excel 文件
- 系统自动识别该文件属于哪个业务模块（员工/工资/返费/考勤/财务等）
- 系统自动将 Excel 列名映射到系统标准字段（适配各种非标准表头命名）

**3.13.2 人工确认与调整**
- 预览 AI 识别结果：目标模块、列映射关系、前几条数据预览
- 用户可手动调整列映射（拖拽或下拉选择）
- 确认后一键执行导入

**3.13.3 值自动转换**
- 自动转换不同格式的值（如"男"→标准值、"在职"→标准值）
- 无法自动转换的值标记为待处理，提示用户手动修正

### 3.14 预警中心

**3.14.1 预警类型与触发条件**

| 预警类型 | 触发条件 |
|---------|---------|
| 未签合同预警 | 员工入职超过 20 天仍未签订合同 |
| 合同到期预警 | 合同到期前 15 天 |
| 回款逾期预警 | 超过预计回款日期仍未到账 |
| 企业资质到期预警 | 企业资质文件即将到期 |
| 数据不一致预警 | 日记账总额与各财务模块汇总不一致 |

**3.14.2 预警展示**
- 统一预警中心页面，集中展示所有类型预警
- 每条预警展示关键信息（涉及人员/企业、触发原因、天数/金额）
- 支持按预警类型筛选
- 预警信息在首页 Dashboard 也可见

### 3.15 数据报表

- 入职统计报表：按企业、月份、岗位维度查看入职人数趋势
- 发薪汇总报表：按月份、企业维度查看用工成本
- 回款统计报表：按月份、企业维度查看回款完成率
- 利润分析报表：按月/季度查看收入、支出、净利润趋势
- 报表以图表形式展示（柱状图、趋势线等）
- 支持导出报表数据

### 3.16 操作日志

- 自动记录所有关键操作：谁、什么时间、在哪个模块、做了什么
- 记录内容包括：操作人、操作时间、操作模块、操作类型（新增/编辑/删除/导出）、操作描述、IP 地址
- 支持按操作人、模块、时间范围筛选
- 日志不可被任何人删除或修改
- 敏感操作（删除数据、修改金额等）需特别标注

### 3.17 三级审批流程

- 业务审批链：员工录入 → 财务审核 → 老板确认
- 适用于：工资发放、返费录入、大额财务记录
- 每级审批记录审批人、审批时间、审批意见
- 数据在未完成全部审批前不生效（或标记为待审批状态）
- 审批被驳回时需填写驳回原因，退回上一级

## 4. 业务规则

### 4.1 数据规范

| 规则 | 说明 |
|------|------|
| 日期格式 | YYYY-MM-DD |
| 月份格式 | YYYY-MM |
| 金额精度 | 精确到分（小数点后 2 位），不得有四舍五入误差 |
| 手机号 | 11 位数字，1 开头 |
| 身份证号 | 18 位（含末位 X） |
| 性别 | 男 / 女 |
| 发票号 | 全局唯一，不可重复 |

### 4.2 业务联动规则

- **工资发放 → 日记账**：工资标记为"已发放"后，自动在日记账新增一条支出记录
- **返费支出 → 日记账**：返费记录创建后，自动在日记账新增一条支出记录
- **回款到账 → 日记账**：回款标记为"已到账"后，自动在日记账新增一条收入记录
- **财务记录 → 日记账**：财务收支记录创建后，自动同步到日记账
- **日记账条目来源可追溯**：每条日记账记录必须标明是自动同步还是手动录入、来源于哪个业务模块
- **联动数据变更**：如果工资/返费/回款/财务记录在同步后被修改，日记账对应条目需标记为"待核对"，触发数据不一致预警

### 4.3 利润核算规则

- 收入 = 当月回款到账总额（所有企业汇总）
- 工资支出 = 当月已发放工资总额（所有员工汇总）
- 返费支出 = 当月返费总额（所有企业汇总）
- 其他支出 = 当月财务记录中的支出总额
- 净利润 = 收入 - 工资支出 - 返费支出 - 其他支出
- 每月只能有一条利润核算记录（如重复计算则覆盖）

### 4.4 状态流转规则

**员工状态**：在职 ↔ 离职（离职后不可恢复为在职？还是允许重新入职？需确认）

**企业合作状态**：正常合作 → 暂停合作 → 终止合作（可逆向，如暂停后可恢复）

**合同状态**：生效中 → 已到期（自动）/ 已终止（手动）

**工资状态**：待发放 → 已发放（不可逆向）

**回款状态**：待回款 → 已到账 / 已逾期（自动判定）

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 多用户协作 | 支持多用户同时在线操作，数据实时同步，权限隔离 |
| 数据一致性 | 财务模块间数据自动联动；不一致时触发提醒 |
| 可追溯性 | 所有操作记录日志，不可删除、不可修改 |
| Excel 兼容 | 所有业务模块支持 Excel 模板下载、批量导入、数据导出 |
| 筛选灵活性 | 所有列表页支持多维度组合筛选 |
| 响应速度 | 列表查询 < 1 秒，利润核算等复杂报表 < 3 秒 |
| 移动适配 | 支持手机和平板访问（响应式布局） |
| 部署简便 | 部署步骤简单，配置清晰 |

## 6. 术语表

| 术语 | 说明 |
|------|------|
| 日结工 | 按日出勤结算的劳务派遣员工 |
| 代招返费 | 为用工企业代招员工后，企业支付的招聘服务费 |
| 直接给付 | 回款方式：现金或转账直接支付 |
| 银行承兑 | 回款方式：银行承兑汇票，有固定到期日，到期后方可兑付 |
| 日记账 | 按日记录所有资金进出的明细账，是财务核算的基础 |
| 利润核算 | 月度汇总所有收入和支出，计算净利润 |

## 7. 不在范围内

以下需求明确排除：

1. 社保、公积金、个税计算
