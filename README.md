# 曼克斯劳务派遣经营管理系统

面向劳务派遣公司的联网业务管理系统。正式业务数据存储在 PostgreSQL；Excel 用于批量收集、清洗、导入和导出。AI 仅作为顾问，不直接修改业务数据。

## 当前功能

- 账号登录、首次管理员初始化、员工/财务/老板三级角色及管理员账号管理
- 企业、岗位、人员、合同的查询、新增、编辑与停用/离职/终止
- 企业与人员 Excel 暂存、清洗、错误预览、确认入库
- 身份证加密存储、手机号/性别/入职日期校验、企业和岗位匹配
- 入职超过 20 天未签合同、合同到期 15 天预警
- 日记账 Excel 清洗入库，收入和支出分开查看，手工增改、作废、筛选和导出
- 人员详情关联展示合同、考勤和工资记录
- 工资、返费、开票、回款的新增、编辑、作废及 Excel 导入导出
- 回款可分配至应收记录；直接给付与银行承兑分别记录，承兑需填写到期日
- 工资与返费经老板确认、回款录入后自动生成日记账并保留来源关联
- 重要增删改操作自动写入 `audit_logs`
- 首页经营摘要、预警和待办入口

## 使用前准备

1. 安装 PostgreSQL，并使用 DBeaver 创建数据库 `manx_hr`。
2. 在 DBeaver 中执行 [backend/sql/schema_v1.sql](backend/sql/schema_v1.sql)。
3. 创建 `backend/.env`，至少配置：

```env
DATABASE_URL=postgresql+psycopg://数据库账号:数据库密码@127.0.0.1:5432/manx_hr
DATA_ENCRYPTION_KEY=Fernet密钥
AUTH_SECRET=长度足够的随机密钥
AUTH_TOKEN_HOURS=12
COOKIE_SECURE=false
```

正式部署 HTTPS 时必须设置 `COOKIE_SECURE=true`。`.env` 不得提交到 Git。

```powershell
..\venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```


浏览器访问 `http://127.0.0.1:5173`。第一次打开时创建管理员账号，以后使用该账号登录。

## 日常使用顺序

1. 在“企业管理”手工新增或导入企业。
2. 在“岗位管理”为企业建立岗位。
3. 在“人员管理”手工新增或导入人员；系统会校验身份证、手机号、性别、日期及企业岗位。
4. 在“合同管理”录入合同；首页会显示未签合同和即将到期预警。
5. 在“日记账”导入 Excel 或手工记账，使用收入/支出筛选核对资金。
6. 在“设置”创建其他账号并分配一级员工、二级财务、三级老板角色。

无需先导入表格也能手工新增。岗位依赖企业，人员依赖企业，合同依赖人员，因此应按上述顺序建立基础资料。

## Excel 导入方法

1. 进入对应模块，点击“导入”。
2. 选择 `.xlsx` 文件和工作表。
3. 点击“整理并校验”。
4. 检查可写入行与阻断原因；错误行不会进入正式数据库。
5. 点击“确认写入数据库”。

人员表建议包含：`姓名、身份证号、手机号、性别、地址、入职日期、企业名称、岗位名称`。

## 验收清单

### 1. 数据库与登录

- 打开 `/api/health/db`，确认 `status=ok` 且数据库为 `manx_hr`。
- 首次进入创建管理员，退出后能重新登录。
- 未登录直接访问业务 API 应返回 `401`。

### 2. 基础资料闭环

- 不导入 Excel，手工新增企业和岗位。
- 手工新增人员，刷新页面后数据仍存在。
- 编辑人员手机号，再到 DBeaver 查询 `employees`，确认已更新。
- 办理离职，确认人员状态及 `employment_records.leave_date` 改变。

### 3. 人员导入清洗

- 准备一行正确数据、一行错误手机号、一行不存在的企业或岗位。
- 正确行应显示“可写入”，错误行应明确显示原因。
- 确认导入后只有正确行进入 `employees` 和 `employment_records`。
- 重复身份证再次导入必须被阻断。

### 4. 合同预警

- 为入职超过 20 天的人员不录合同，应出现未签合同预警。
- 录入 15 天内到期的合同，应出现到期预警。
- 终止合同后状态应变为 `terminated`。

### 5. 日记账

- 导入同时含收入和支出的 Excel。
- 分别选择“收入”和“支出”，核对金额与行数。
- 手工新增、编辑、作废一条记录，刷新后结果保持一致。
- 导出 Excel，核对筛选条件和金额。

### 6. 权限与追溯

- 管理员在“设置”创建一级员工、二级财务、三级老板账号。
- 普通账号不能进入账号管理接口。
- 完成一次新增或修改，在 DBeaver 查询 `audit_logs`，确认操作者、时间、路径和动作存在。

### 7. 自动化检查

```powershell
cd D:\人力资源项目\backend
..\venv\Scripts\python.exe -m pytest -q

cd D:\人力资源项目\frontend
npm run build
npm run lint
```

预期：后端测试全部通过，前端构建和检查无错误。

连接真实 PostgreSQL 的只读接口冒烟：

```powershell
cd D:\人力资源项目\backend
..\venv\Scripts\python.exe -m scripts.smoke_real_db
```

预期所有模块均显示 `PASS`。

## 上线前必须完成

当前版本适合本机或受控内网试用，不应直接暴露到公网。公网生产上线前仍需完成：

- 财务 Excel 模板兼容性和大批量真实数据压力验收
- 利润核算、日记账联动、自动核对与差异提醒
- PostgreSQL 自动备份、恢复演练和附件备份
- HTTPS、生产域名、反向代理、`COOKIE_SECURE=true` 和 CORS 白名单
- 账号停用、密码修改/重置、登录限流和安全审计
- 多用户并发、权限矩阵、真实 Excel 和全年度数据验收

在这些项目全部验收通过前，不要将系统称为正式生产版。
